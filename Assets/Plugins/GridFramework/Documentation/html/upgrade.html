Grid Framework  2.1.4
Easy grid setup and scripting for Unity 3D
Upgrade Guide

Grid Framework version 2.0 is a major rewrite of the previous version 1.x releases. Being a major version number increase, this release has broken compatibility with the version 1.x API. This chapter is intended for users of previous releases who wish to make their code work with version 2.0 and higher.

Note
This chapter replaces the old "legacy support" chapter of the version 1.x manual

Changed Features

Namespaces instead of prefixes

Version 1.x was targeted at a version of Unity that had flawed support for custom namespaces. This is no longer the case and we can use namespaces to avoid name collision. You have to remove every GF prefix from class names and instead import the proper namespace of the class. For example GFRectGrid becomes just RectGrid and you have to import the GridFramework.Grids namespace.

Do not use Grid anymore

Version 1.x encouraged the use of the GFGrid class (now renamed Grid) by having a number of public methods and properties in it. This turned out fairly useless and after the cleanup there is nothing public left anymore. This makes references to Grid useless in most cases. Grids just aren't polymorphic enough.

Extension methods

Some methods have been made into extension methods rather than being part of the grid classes themselves. These are methods which do not serve the task of "being a grid" and which are efficiently implemented using only the public methods and properties provided by their classes.

Nearest and alignment methods

The NearestVertex method is one example of a method that has been factored out (NearestBox has been renamed NearestCell). NearestFace only exists for the layered grids now, for the others you will have to write it as a combination of the results from NearestVertex and NearestCell.

Alignment methods are only defined for rectangular and hexagonal grids because that's where they make the most sense. Scale methods are only defined for rectangular grids.

Rendering tasks have been factored out

Anything related to rendering and drawing has been removed from the grids and been made its own class. If your code relies on the renderFrom of a grid you have to instead reference the From of a renderer instead.

Renderers have their own notion of what a range is. In version 1.x all grids shared a renderFrom and renderTo property, but this universal data did not make sense for all grids. In version 2.0 the data depends on the type of renderer.

GridChangedDelegate and events

This type has been replaced in favour of more specific delegates and events for individual properties.

Moved into the renderer classes

The following properties and methods have been moved from the grids to the renderers.

renderMaterial is now the Material property of renderers, renderLineWidth is LineWidth, and axisColors is ColorX, ColorY and ColorZ.

Removed features

Some features have been removed completely. These were features that served no practical purpose when they can be easily implemented in a different way just as efficiently.

GFColorVector3 and GFBoolVector3

These two classes have been removed entirely without any replacement. GFColorVector3 had been used only in one instance where it has now been replaced by three individual Color properties. GFBoolVector3 is no loner needed.

RenderGrid and DrawGrid

These have been removed completely in favour of the new rendering pipeline.

useSeparateRenderColor and renderAxisColors

You can set separate colours instead at runtime using scripting by setting the values of ColorX, ColorY and ColorZ.

hideGrid, hideOnPlay and hideAxis

When you set the alpha value of a colour to zero that axis will not get rendered. You can disable all rendering by disabling the renderer component itself.

size, relativeSize and useCustomRenderRange

These are leftover from very early releases; size was like setting renderFrom and renderTo to opposite values, so set your renderer's range accordingly. Whether sizes are relative depends on the renderer (they are in all the official ones).

originOffset

You should make your grid object the child of another object and move it accordingly.

GFGridPlane type and plane of layered grids

Layered grids no long have a plane, instead they always default to an XY-plane. You should instead rotate your grid the way you want it to be.
